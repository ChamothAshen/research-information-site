Research & Technology

Domain

Research background, problem statement, and technical approach

Literature Survey

Existing research in tourism management systems has largely focused on static data analysis and manual monitoring approaches. Early works by Gössling et al. (2018) examined the environmental impact of mass tourism on heritage sites, identifying the need for real-time monitoring systems.

Computer vision applications in crowd analysis have advanced significantly with the introduction of deep learning models. Studies employing YOLO-based architectures have demonstrated impressive accuracy in pedestrian detection in complex outdoor environments (Redmon & Farhadi, 2018).

Predictive analytics for tourism demand has evolved from traditional ARIMA models to modern deep learning and probabilistic forecasting approaches. Facebook's Prophet model has shown particular effectiveness for tourism data with strong seasonal patterns and holiday effects.

Existing smart tourism platforms such as those deployed in Barcelona, Amsterdam, and Singapore have demonstrated the value of digital twin technology, yet no comprehensive implementation exists for heritage sites in South Asia — particularly those with unique topographic challenges like Sigiriya.

Research Gap

A thorough analysis of existing solutions reveals several critical gaps that this project aims to address:

  • Gap 1: No integrated real-time crowd management + forecasting system exists for UNESCO World Heritage Sites in Sri Lanka
  • Gap 2: Existing environmental risk systems do not account for topographic-specific hazards (steep terrain, weather, crowd density synergies)
  • Gap 3: No mobile-first tour guidance app integrates live crowd data, risk alerts, and path recommendations simultaneously
  • Gap 4: Current visitor forecasting tools lack 90-day horizon accuracy sufficient for operational planning

Research Problem Statement

"How can we efficiently manage real-time visitor flow, forecast demand, assess environmental safety risks, and guide tourists intelligently at Sigiriya Rock Fortress through an integrated AI-powered digital twin platform?"

Sigiriya Rock Fortress receives over 500,000 visitors annually. The site faces critical challenges:

  • ◆ Dangerous overcrowding at narrow passages and stairways during peak hours
  • ◆ Unpredictable visitor numbers making resource allocation extremely difficult
  • ◆ Environmental risks from weather, erosion, and infrastructure stress going undetected
  • ◆ Lack of intelligent guidance causing visitor congestion at popular areas

Research Objectives

  • 1Develop a YOLOv8-based real-time crowd detection and density mapping system covering 11 key points of interest at Sigiriya
  • 2Build a 90-day visitor forecasting REST API using Meta Prophet time-series model trained on historical tourism data
  • 3Create a multi-layer environmental and safety risk prediction system integrating weather, crowd, and historical incident data
  • 4Design and deploy an intelligent path prediction algorithm for optimal tourist route recommendations
  • 5Build a cross-platform Flutter mobile application integrating all modules for visitor and management use
  • 6Integrate all sub-systems into a unified digital twin platform with FastAPI backend and MongoDB data persistence

Methodology

The project follows a modular, iterative development methodology with five distinct phases:

Phase 1
Data Collection & Preprocessing — Historical visitor data collection from SLAS, CCTV footage annotation, weather data integration from meteorological services, and site mapping
Phase 2
Model Development — YOLOv8 fine-tuning on custom dataset, Prophet model training with hyperparameter optimization, risk classification model development using Scikit-learn ensemble methods
Phase 3
API Development — FastAPI backend with MongoDB integration, RESTful endpoints for all modules, real-time WebSocket connections for live data streaming
Phase 4
Mobile Application — Flutter cross-platform app development with interactive map, real-time alerts, tour guidance, and admin dashboard modules
Phase 5
Integration & Deployment — System integration testing, Railway cloud deployment, performance optimization, and live pilot testing at Sigiriya
